# Architecture

## What This Is

A company-wide AI configuration marketplace serving two outputs from one filesystem source:

1. **Copilot CLI marketplace** — `copilot plugin marketplace add danforceDJ/ai-catalog`
2. **Static web marketplace** — GitHub Pages with live search and four install paths per card

| Standard | Purpose |
|---|---|
| [AGENTS.md](https://agents.md/) | Agent behavior instruction files |
| [Agent Skills](https://agentskills.io/) | Reusable skill packages via `SKILL.md` |
| [Model Context Protocol](https://modelcontextprotocol.io/) | MCP server configs via `.mcp.json` |
| [Copilot CLI plugin format](https://docs.github.com/en/copilot/how-tos/copilot-cli/customize-copilot/plugins-creating) | Native install via `copilot plugin install` |

## Data Flow

```
plugins/  +  skills/  +  agents/
prompts/  +  mcp/  +  templates/
system/config/marketplace.config.json
      │
      ▼
  generate_catalog.py      →  system/artifacts/catalog.json  (tracked)
  generate_marketplace.py  →  .github/plugin/marketplace.json  (tracked)
                               plugins/<name>/.copilot-plugin/  (generated)
      │
      ▼
  generate_site.py         →  docs/index.html  (not committed)
  generate_zips.py         →  docs/dl/<name>.zip  (not committed)
      │
      ▼
  GitHub Pages + Copilot CLI
```

## Tracked vs Generated Files

| File | Committed | Regenerated by |
|---|---|---|
| `system/artifacts/catalog.json` | **Yes** | `generate_catalog.py` — run locally + commit with PR |
| `.github/plugin/marketplace.json` | **Yes** | `generate_marketplace.py` — run locally + commit with PR |
| `docs/index.html` | No | CI deploy workflow only |
| `docs/dl/*.zip` | No | CI deploy workflow only |

## Two-Tier Content Model

### Primitives — standalone, shareable

| Type | Path | File |
|---|---|---|
| Skill | `skills/<name>/` | `SKILL.md` |
| Agent | `agents/` | `<name>.agent.md` |
| Command | `prompts/` | `<name>.md` |
| MCP Server | `mcp/<name>/` | `.mcp.json` |
| Template | `templates/<name>/` | `TEMPLATE.md` |

Discoverable in the web UI. Not directly Copilot CLI installable unless wrapped in a plugin.

### Plugins — Copilot CLI installable

Every plugin lives at `plugins/<name>/plugin.json`. References primitives by name list:

```json
{
  "name": "my-bundle",
  "skills": ["skill-a"],
  "mcpServers": ["my-mcp"]
}
```

Derived type from component composition:

| Components in `plugin.json` | Type |
|---|---|
| Only skills | `skill` |
| Only agents | `agent` |
| Only commands | `prompt` |
| Only MCP servers | `mcp` |
| ≥ 2 component kinds | `bundle` |
| None | `empty` |

## `.copilot-plugin/` Package Generation

`generate_marketplace.py` creates `plugins/<name>/.copilot-plugin/` for list-reference plugins:
1. Copies primitive files in (`SKILL.md`, `*.agent.md`, `*.md`, `.mcp.json`)
2. Merges multiple MCP server configs into one `.mcp.json`
3. Converts list refs → string paths in the output `plugin.json`
4. Sets `source` in `marketplace.json` to `.copilot-plugin/`
