# Install Paths

Four install methods are available per catalog item, depending on type.

## 1. Copilot CLI (primary)

**Available for:** All 5 plugins (not standalone primitives, not templates)

```bash
# Register the marketplace once
copilot plugin marketplace add danforceDJ/ai-catalog

# Install a specific plugin
copilot plugin install <name>@ai-catalog
```

The Copilot CLI reads `.github/plugin/marketplace.json`, resolves the `source` path, and installs from it.

## 2. VSCode MCP Deeplink

**Available for:** Plugins that contain MCP servers, when the base64url-encoded config payload is ≤ 2048 bytes.

```
vscode:mcp/install?name=<server-name>&config=<base64url-encoded-json>
```

Clicking opens VSCode directly and installs the MCP server config. Generated by `build_deeplink()` in `generate_catalog.py`.

## 3. Copy Raw File (web modal)

**Available for:** Single-primitive string-path plugins (agent, skill, command) and all templates.

The web UI fetches:
```
https://raw.githubusercontent.com/danforceDJ/ai-catalog/main/<repoPath>/<file>
```

Displayed in a `<dialog>` modal with a "Copy" button.

## 4. Download `.zip`

**Available for:** All plugins and templates.

Direct link to `docs/dl/<name>.zip` — a ZIP of the entire plugin or template directory.

> **Note:** Plugin ZIPs currently include the generated `.copilot-plugin/` sub-directory.

---

## Install Path Matrix

| Item type | Copilot CLI | VSCode deeplink | Copy raw | Download zip |
|---|---|---|---|---|
| Plugin (mcp) | ✅ | ✅ (if ≤ 2KB) | ❌ | ✅ |
| Plugin (skill/agent/prompt) | ✅ | ❌ | ✅ (string-path only) | ✅ |
| Plugin (bundle) | ✅ | ✅ (if has MCP, ≤ 2KB) | ❌ | ✅ |
| Standalone primitive | ❌ | ❌ | ✅ | ❌ |
| Template | ❌ | ❌ | ✅ | ✅ |

## Legacy `install.sh` (deprecated)

Still ships in the repo but prints a deprecation banner. MCP installs merge `servers` objects into `.vscode/mcp.json` (or JetBrains IDE config dirs) using `jq`. See [`scripts.md`](scripts.md) for known issues.
